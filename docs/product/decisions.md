## ADR-001：单用户模式，省略user_id字段

**日期**：2026-03-13

### Problem
多用户系统需要user_id隔离，增加复杂度。当前仅服务单一用户。

### Decision
所有表省略user_id字段，直接查询。

### Rationale
- 减少90%查询条件代码
- 简化索引设计
- 单用户场景无需隔离

### Trade-offs
- 未来扩展多用户需重构（可接受，MVP优先）
- 代码层面无法防止误操作（通过测试覆盖）

## ADR-002：飞书表格作为展示层主载体

**日期**：2026-03-13

### Problem
需要可视化展示训练记录、日历、Dashboard。自研前端成本高。

### Decision
复用飞书原生能力：多维表格 + 日历 + 仪表板。

### Rationale
- 零前端开发成本
- 用户已在飞书生态
- 双向编辑天然支持
- 权限控制飞书已解决

### Trade-offs
- 可视化样式受限（飞书固定模板）
- 高级图表需后端计算后写入
- 依赖飞书API稳定性

## ADR-003：离线同步策略 - 乐观锁+时间戳优先

**日期**：2026-03-13

### Problem
用户可能在飞书离线编辑，后端同时有数据变更，产生冲突。

### Decision
乐观锁版本控制 + 时间戳优先（LWW）+ 冲突时人工确认。

### Rationale
- 单用户场景冲突概率低
- 时间戳规则简单易懂
- 极端冲突可人工介入

### Trade-offs
- 需要额外version字段
- 冲突时需用户操作（但频率低）
- 无自动合并策略（简单优先）

**日期**：2026-03-13

### Problem
训练APP截图需要提取结构化数据（动作/组数/重量）。

## ADR-005：AI角色系统 - 自动路由

**日期**：2026-03-13

### Problem
用户需求多样：计划生成、康复指导、动作替代，需要不同专业视角。

### Decision
三角色自动路由：训练师/康复师/运动科学顾问。

### Rationale
- 用户无需手动选择，降低认知负担
- 关键词触发逻辑简单可靠
- 上下文继承保持对话连贯

### Trade-offs
- 需要维护角色切换规则表
- 混合场景需多角色协作（复杂度+1）
- 误诊风险（关键词误判）

## ADR-006：项目目录结构 - 单仓库统一管理

**日期**：2026-03-14

### Problem
用户要求将开发服务和AI角色（健身教练）放在一个openclaw里管理，不拆分成独立项目。

### Decision
采用单仓库（Monorepo）结构，后端服务和AI角色统一放在 `backend/` 目录下：
- `backend/services/ai/` - 训练师、康复师等AI角色
- 整个项目后续同步到GitHub一个仓库

### Rationale
- 开发阶段减少仓库管理复杂度
- AI角色和后端服务紧密耦合（共享数据模型）
- 单用户场景，无需微服务拆分

### Trade-offs
- 后期如需独立部署AI服务需重构（当前可接受）
- 代码库会包含Prompt和业务逻辑混合（通过目录隔离）

### 目录结构
```
fitness-assistant/
├── backend/                 # FastAPI后端服务
│   ├── api/                 # REST API端点
│   ├── models/              # SQLAlchemy模型（共享）
│   ├── services/            # 业务逻辑
│   │   ├── ai/              # AI角色（训练师/康复师/顾问）
│   │   │   ├── trainer.py   # 训练师角色
│   │   │   ├── rehab.py     # 康复师角色
│   │   │   └── router.py    # 角色路由
│   │   ├── parser.py        # 文本解析
│   │   ├── ocr.py           # OCR识别
│   │   └── feishu.py        # 飞书集成
│   └── main.py              # 服务入口
├── scripts/                 # 数据库初始化等脚本
├── docs/                    # 项目文档
└── docker-compose.yml       # 部署配置
```
